## 4. ארבעת השלבים

### שלב 1: פרטי פרויקט
- שם פרויקט (אופציונלי)
- שם משרד/גוף (אופציונלי)
- תקופת חישוב: 6 / 12 / 24 חודשים (SegmentedControl, ברירת מחדל: 12) — **3 חודשים הוסר** כי אין לו KPI card מתאים בשלב 4
- סוג תעריף: תעריף גג 🔴 (פעיל) / תעריף מרבי 🟡 (מנוטרל, disabled עם tooltip: "יתווסף בגרסה 2 — ממתין לנתונים")
- Toggle: מאצ'ינג — מערך הדיגיטל הלאומי (כן/לא, ברירת מחדל: כן)
- אחוז מאצ'ינג: שדה מספרי 1–100 (ברירת מחדל: 30%, מוצג רק כשהtoggle פעיל)

### שלב 2: בחירת תפקידים
- חיפוש חופשי בשם תפקיד
- פילטר קטגוריות (pills): הכל / ניהול / פיתוח / תשתיות / תמיכה / דאטה ובינה / ERP/SAP / ארכיטקטורה
  - הקטגוריה "דאטה ובינה" כוללת: תפקידים 2.7 (אנליטיקה), 2.11 (מדע הנתונים), 2.12 (הנדסת נתונים)
- רשימת תפקידים — כרטיסיות לחיצות (toggle select)
- סכירת נבחרים בתחתית המסך
- "המשך" מופעל רק אם נבחר לפחות תפקיד אחד

### שלב 3: הגדרת תמהיל
- לכל תפקיד שנבחר:
  - כרטיס עם שם + קטגוריה
  - בחירת רמת מומחיות (א/ב/ג/ד) — כפתורים; רמה לא-קיימת לתפקיד מסומנת כ-N/A ומנוטרלת; מציג תעריף (₪/שעה) לכל רמה
  - Slider אחוז משרה: 10%–100%, קפיצות של 10%
  - תצוגת עלות חודשית משוערת בזמן-אמת (ראה לוגיקת חישוב)
  - כפתור "הסר" להוצאה מהתמהיל
- ניתן לחזור לשלב 2 ולהוסיף/להסיר תפקידים

### שלב 4: תוצאות ועלויות
- 3 כרטיסיות KPI: 6 חודשים / 12 חודשים / 24 חודשים; כרטיסייה הרלוונטית לתקופה שנבחרה מודגשת
- טבלת פירוט לפי תפקיד (שם, רמה, אחוז משרה, עלות שנתית)
- גרף עמודות אופקיות — התפלגות עלות לפי תפקיד
- גרף Stacked bar — ברוטו vs. מאצ'ינג (מוצג רק כשמאצ'ינג פעיל)
- כפתורי פעולה: הורד PDF | שתף קישור | ערוך תמהיל | פרויקט חדש
  - **הורד PDF**: `window.print()` עם `@media print` CSS מותאם — ללא ספריות חיצוניות (עקב קובץ יחיד)

## 5. לוגיקת חישוב

### נוסחת בסיס
```
שעות_חודשיות  = 176 × (אחוז_משרה / 100)        // 22 ימי עבודה × 8 שעות
עלות_חודשית   = תעריף_שעתי × שעות_חודשיות × 1.17  // כולל מע"מ
עלות_תפקיד   = עלות_חודשית × מספר_חודשים
עלות_כוללת   = סכום עלות_תפקיד לכל תפקידים בתמהיל
עלות_משרד    = עלות_כוללת × (1 − אחוז_מאצ'ינג)   // אם מאצ'ינג פעיל
```

### מע"מ
```js
const VAT = 1.17; // 17% — קבוע; לא מחושב דינמי
```
מע"מ חל על כל תפקידי ייעוץ חיצוני. מוצג בנפרד בפירוט.

### תעריף מרבי לעומת תעריף גג
- **תעריף גג** = המחיר המרבי המוחלט (₪/שעה) — **v1 מחשב רק לפי זה**
- **תעריף מרבי** = תעריף רכש מוגבל נמוך יותר — **v2 בלבד**, ממתין לנתונים מהאקסל
- כפתור "תעריף מרבי" בשלב 1 מנוטרל (disabled) עם tooltip: "יתווסף בגרסה 2"

### הנחות חישוב
- **176 שעות/חודש** = 22 ימי עבודה × 8 שעות — ממוצע קבוע לצרכי תכנון, לא מדויק ליומן ספציפי
- מספרי תפקיד חסרים (1.4, 2.10, 3.5, 3.6, 6.3) — לא קיימים בנספח יג המקורי; אין להתייחס אליהם כשגיאת נתונים

### פורמט hash לשיתוף
```
#v1|period=12|tariff=ceiling|matching=1|pct=30|roles=1.6:b:100,2.3:c:50,3.4:b:75
```
- `period` — מספר חודשים (6/12/24)
- `tariff` — ceiling בלבד בv1
- `matching` — 0/1
- `pct` — אחוז מאצ'ינג
- `roles` — רשימה מופרדת בפסיק; כל תפקיד: id:רמה:scope%

### עיגול ועיצוב מספרים
- כל סכומים: Math.round לשקל שלם
- תצוגה מקוצרת בKPI: K לאלפים (₪459K), M למיליונים (₪1.2M)
- תעריפי שעה: מספר שלם (₪202)

## 8. ארכיטקטורה — קובץ יחיד

```
takam-calculator.html
├── <style>  — CSS inline (Heebo, design tokens, responsive)
├── <body>
│   ├── topbar
│   ├── wizard-progress (steps 1–4)
│   └── main
│       ├── panel#p1 — פרטי פרויקט
│       ├── panel#p2 — בחירת תפקידים
│       ├── panel#p3 — הגדרת תמהיל
│       └── panel#p4 — תוצאות
└── <script>
    ├── ROLES_DATA[]   — כל 36 תפקידים עם תעריפים
    ├── state{}        — מצב הטופס (project, selectedRoles, mix, results)
    ├── goStep(n)      — מעבר בין שלבים + validation
    ├── renderMix()    — בניית כרטיסי שלב 3 דינמית
    ├── calcResults()  — חישוב עלויות
    ├── renderResults()— עדכון DOM שלב 4
    ├── renderCharts() — גרפי SVG inline
    ├── shareURL()     — קידוד state ב-hash URL
    └── loadFromURL()  — שחזור state מ-hash
```
